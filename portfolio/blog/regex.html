What are Regular Expressions?

A regular expression (regex) is a pattern; it’s a sequence of characters (a string) that describes a search pattern. Regular expressions are most commonly used to search text and validate input. This post will explore how metacharacters, character classes, and lookaheads/lookbehinds represent text in a regular expression.

Metacharacters

Metacharacters such as `.`, `?`, and `$` have a special meaning inside regular expressions.

Description Example Match Non Match
. Any letter, digit, symbol, or space c.po c3po, c@po, c po cap, cappuccino
? 0 or 1 times colou?r color, colour colur
* 0 or more times 10*1 11,101,100001 001
+ 1 or more times 10+1 101,100001 11
^ Beginning of string ^N N, Nova “ N”, nova
$ End of string a$ Nova, n o v a “nova ”, NOVA

Metacharacters must be escaped with a backslash to represent themselves inside a regular expression; “\” also must be escaped.

Example: Price

^\$\d+\.\d{2}$

`\d+` matches at least one digit. `\d{2} matches exactly two digits. The first dollar sign is escaped, `\$` to match literally and the last dollar sign is not escaped to match the end of the string. The dot is escaped, `\.` to match a dot instead of any character.

Or Statements

Description Example Match Non Match
(a|b|c) a or b or c gr(a|e)y gray, grey gry, greay
[abc] a or b or c seriali[sz]e serialise, serialize serialie, serializse
[^abc] not a or b or c [^hb]at cat, mat hat, bat

Note that `^` on it’s own represents the start of the string while `^` inside of square brackets negates the `or` statement and matches anything not included in the brackets. Meta character become literal inside of a set and escaping them is not necessary.

Example: Password Contains a Special Character

[!@#$%^&*.(),-]

None of these characters need to be escaped inside a set. Hyphen is a special case and must be the beginning or end of the set. A hyphen between two characters will match the range between those characters.

[ae-] matches `a`, `e`, and `-`.

[a-e] matches `a`,`b`,`c`,`d` and `e`, and does not match `-`.

Ranges & Character Classes

Ranges, character class and shorthand class notation are simple ways to match many similar characters at once.

All pattern are not usable in all instances, for example, POSIX character classes are not valid in JavaScript or when using Python's `re` module.

Range POSIX Shorthand
Whitespace [:space:] \s
Not whitespace ^[:space:] \S
Letter, Digit, or Underscore [A-Za-z0-9_] [:word:] \w
Not Letter, Digit, or Underscore [^A-Za-z0-9_] ^[:word:] \W
Digit [0-9] [:digit:] \d
Not Digit [^0-9] ^[:digit:] \D
Uppercase [A-Z] ^[:upper:] \u
Lowercase [a-z] ^[:lower:] \l
Letter [A-Za-z] [:alpha:]
Not Letter [^A-Za-z] ^[:alpha:]
Punctuation [!-/:-@[-`{-~] [:punct:]
Word boundary [:<:] & [:>:] \b

A range can include all letters `[A-Za-z]` or just a few `[A-E]`. Ranges match all ASCII characters, not just letters and numbers, so the expression `[!-+]` matches the characters with ASCII codes between `!` and `+` which are `!”#$%&’()*+`.

Counting Characters

Description Example Match
{min,} At least min in a row \d{2,} 42, 42424242
{min,max} Between min - max in a row \d{2,4} 42, 424, 4242
{n} exactly n in a row \d{2} 42

Adding counts to a regular expression can highlight the importance of making the pattern as specific as possible. The regular expression `\d{6}` will match six digit numbers. However, without context before and/or after it will also match any number with more than 6 digits.

Example: Phone Number

^\(\d{3}\)\s\d{3}-\d{4}$

`\d{4}` without`$`matches `1234567890` because we didn't specify that the four digits not be followed by six more digits.

Lookarounds

Lookahead and Lookbehind assertions check the pattern before and after the current location. A `positive` lookaround matches the pattern and a `negative` lookaround excludes the pattern.

Pattern Description Example Match Non Match
(?=pattern) Positive lookahead a(?=b) ab aa
(?!pattern) Negative lookahead a(?!b) aa ab
(?<=pattern) Positive lookbehind (?<=b)a ba ab
(?<!pattern) Negative lookbehind (?<!b)a aa ba

Example: Validate Username

Let's write a regular expression to validate a username. In our system, valid usernames may contain:

  • 6 - 12 characters
  • Uppercase and lowercase letters
  • Digits
  • Special Characters: !#$%&*+-=?^_~
  • Dot - NOT as first or last character and not consecutively

There are many working solutions to this problem. This solution incorporates lookaround assertions, a set, a character count, and a character class shorthand.

^(?!\.)(?!.*\.\.)[\w!#$%&*+-=?^~]{6,12}(?<!\.)$

^(?!\.) is a negative lookahead. `^` matches the beginning of the string, but only if it is not followed by `.` so, any string that doesn't start with `.` matches this section of the pattern.

(?!.*\.\.) is a zero-width negative lookahead assertion. Unlike other lookahead assertions we've seen this one doesn't match a single character, `.*` matches every character after the assertion until the end of the string; `\.\.` checks that no character is followed by consecutive dots.

[\w!#$%&*+-=?^~]{6,12} is a set that takes care of matching all the non dot valid characters. `\w` matches all digits, letters and underscore; each individual character in `!#$%&*+-=?^~` represents itself, metacharacters inside a set don't need to be escaped. `{6,12}` matches a minimum of six and maximum of 12 characters.

(?<!\.)$ Is a negative lookbehind that matches `$` the end of the string, but only if it is not proceeded by `.`.
